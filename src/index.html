GIT CHEAT SHEET

SETUP-VSCode

Configuring user information used across all local repositories

  1. set a name that is identifiable for credit when review version history
    2. git config --global user.name “[firstname lastname]”
  2. set an email address that will be associated with each history marker
    3. git config --global user.email “[valid-email]”
  3. set automatic command line coloring for Git for easy reviewing
    4. git config --global color.ui auto

      SETUP & INIT

      Configuring user information, initializing and cloning repositories

      1. initialize an existing directory as a Git repository
        2. git init
      2. retrieve an entire repository from a hosted location via URL
        3. git clone [url]

      STAGE & SNAPSHOT

      Working with snapshots and the Git staging area

      1. show modified files in working directory, staged for your next commit
        2. git status
      2. add a file as it looks now to your next commit (stage)
        3. git add [file]
      3. unstage a file while retaining the changes in working directory
        4. git reset [file]
      4. diﬀ of what is changed but not staged
        5. git diff
      5. diﬀ of what is staged but not yet committed
        6. git diff --staged
      6. commit your staged content as a new commit snapshot
        7. git commit -m “[descriptive message]”

      BRANCH & MERGE

      Isolating work in branches, changing context, and integrating changes

      1. list your branches. a * will appear next to the currently active branch
        2. git branch
      2. create a new branch at the current commit
        3. git branch [branch-name]
      3. switch to another branch and check it out into your working directory
        4. git checkout
      4. merge the specified branch’s history into the current one
        5. git merge [branch]
      5. show all commits in the current branch’s history
        6. git log
      6. Second Way to Git log
        7. git log --pretty=oneline
      7. Combining Step 2 and Step 6 in one Command 
        8. git commit -am "commit message"

      INSPECT & COMPARE

      Examining logs, diﬀs and object information

      1. show the commit history for the currently active branch
        2. git log
      2. show the commits on branchA that are not on branchB
        3. git log branchB..branchA
      3. show the commits that changed file, even across renames
        4. git log --follow [file]
      4. show the diﬀ of what is in branchA that is not in branchB
        5. git diff branchB...branchA
      5. show any object in Git in human-readable format
        6. git show [SHA]

      TRACKING PATH CHANGES

      Versioning file removes and path changes

      1. delete the file from project and stage the removal for commit
        2. git rm [file]
      2. change an existing file path and stage the move
        3. git mv [existing-path] [new-path]
      3. show all commit logs with indication of any paths that moved
        4. git log --stat -M

      IGNORING PATTERNS

      Preventing unintentional staging or commiting of files

      1. Save a file with desired patterns as .gitignore with either direct string matches or wildcard globs.
        2. 
  logs/
  *.notes
  pattern*/
      2. system wide ignore pattern for all local repositories
        3. git config --global core.excludesfile [file]

      SHARE & UPDATE

      Retrieving updates from another repository and updating local repos

      1. add a git URL as an alias
        2. git remote add [alias] [url]
      2. fetch down all the branches from that Git remote
        3. git fetch [alias]
      3. merge a remote branch into your current branch to bring it up to date
        4. git merge [alias]/[branch]
      4. Transmit local branch commits to the remote repository branch
        5. git push [alias] [branch]
      5. fetch and merge any commits from the tracking remote branch
        6. git pull [repository link]
